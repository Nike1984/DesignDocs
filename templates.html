Modal Popups

Modal confirmation dialog

While it might be fine to use standard Javascript alerts for development their use for release is discouraged. Instead we should use modal dialogs.

Sample Modal Dialog

Modal dialogs' body

Simple confirmation dialog

The easiest way to add a confimration for an event in .Net is to create a standard clientside control that displays a modal popup and make the confirmation button a server side control that will be used to trigger an event.

Delete

Sample Modal Dialog

Are you sure you want to delete this?

<a href="#modalDialogSampleDelete" role="button" data-toggle="modal" class="btn btn-danger">Delete</a>

<div id="modalDialogSampleDelete" class="modal hide fade" tabindex="-1" role="dialog" aria-labelledby="ModalSample" aria-hidden="true">
	<div class="modal-header">
		<button type="button" class="close" data-dismiss="modal" aria-hidden="true">×</button>
		<h4 id="ModalSample">Sample Modal Dialog</h4>
	</div>
	<div class="modal-body">
		<p>Are you sure you want to delete this?</p>
	</div>
	<div class="modal-footer">.
		<asp:LinkButton ID="btnDelete" runat="server" CssClass="btn btn-primary" data-loading-text="Processing..." Text="Delete Division" />
		<button class="btn btn-danger" style="width:100px">Delete</button>
		<button class="btn btn-inverse" data-dismiss="modal" aria-hidden="true" style="width:100px">Close</button>
	</div>
</div>

Confirmation dialog inside of the repeater control

Often times confirmation dialogs need to be added to each of the element inside of your repeater control (gridview, repeater, datagrid, etc.). This is slightly more complicated. We have added a helper class confirmDialog(source, target) to the utils.js that we can use.

# Element Command
1 Element #1 Delete
2 Element #2 Delete
3 Element #3 Delete
4 Element #4 Delete

Repeater confirmation dialog

Are you sure you want to delete this element?

<!-- Gridview Control -->
<asp:GridView ID="gridviewExample" CssClass="table table-striped table-hover" runat="server" Width="100%" GridLines="None" AutoGenerateColumns="False" >
	<Columns>
		<asp:TemplateField HeaderStyle-CssClass="tablehead leftcell" HeaderText="#">
			<ItemTemplate>
				<asp:Literal ID="ltNumber" runat="server" />
			</ItemTemplate>
		</asp:TemplateField>
		<asp:TemplateField HeaderText="Element" HeaderStyle-CssClass="tablehead midcell" >
			<ItemTemplate>
				<asp:Literal ID="ltElement" runat="server" />
			</ItemTemplate>
		</asp:TemplateField>
		<asp:TemplateField HeaderText="Command" HeaderStyle-CssClass="tablehead rightcell">
			<ItemTemplate>
				<asp:LinkButton ID="btnDelete" runat="server" CommandName="Delete" />
				<a href="#" class="btn btn-danger" onclick="confirmDialog(this, '#modalDialogRepeaterDelete');">Delete</a>
			</ItemTemplate>
		</asp:TemplateField>
	</Columns>
</asp:GridView>
<!-- /Gridview Control -->

		<p>Are you sure you want to delete this element?</p>

Modal with iframe

Including iframe in your modal popup is simple! Just add the <iframe> element with the width attribute set to 100% and desired height and you should be all set.

Show Modal Popup

Sample iframe

<a href="#modalWithIframe" class="btn btn-primary" data-toggle="modal">Show Modal Popup</a>

<div id="modalWithIframe" class="modal hide fade" tabindex="-1" role="dialog" aria-labelledby="ModalSample" aria-hidden="true">
	<div class="modal-header">
		<button type="button" class="close" data-dismiss="modal" aria-hidden="true">×</button>
		<h4 id="ModalSample">Sample iframe</h4>
	</div>
	<div class="modal-body">
		<iframe src="sampleIframe.html" width="100%" height="300" seamless="seamless"></iframe>
	</div>
</div>

Modal with Asp.Net User Control

Alternatively a user control might be included in place of the iframe. To do that we would need to add the user control to the body of the modal popup and trigger the modal display in our code-behind

Add the modal popup control in the page markup 

<div id="modalWithUserControl" class="modal hide fade" tabindex="-1" role="dialog" aria-labelledby="ModalSample" aria-hidden="true">
	<div class="modal-header">
		<button type="button" class="close" data-dismiss="modal" aria-hidden="true">×</button>
		<h4 id="ModalSample">Sample iframe</h4>
	</div>
	<div class="modal-body">
		<uc1:wcSampleUserControl ID="ucSample" runat="server" />
	</div>
</div>

In your code behind - initiate the user control and display the popup

Public Sub ShowPopup()
	'Init control here
	ucSample.visible = True
	ScriptManager.RegisterStartupScript(Me, Me.GetType(), "displayModal", "$('modalWithUserControl').modal('show');", True)
End Sub

Forms

While we can still add forms the traditional way (tables) bootstrap contains a series of classes that make adding good looking forms really easy.

Basic form

The basic form consists of the collection of div elements with the class control-group nested inside of the container with the form-horizontal class. 

<div class="form-horizontal">
	<div class="control-group">
		<label class="control-label" for="<%=txtLogin.ClientID%>">Login</label>
		<div class="controls">
			<asp:TextBox ID="txtLogin" runat="server" CssClass="input-large mandatory" placeholder="Login" />
		</div>
	</div>
	<div class="control-group">
		<label class="control-label" for="<%=txtPassword.ClientID%>">Password</label>
		<div class="controls">
			<asp:TextBox ID="txtPassword" runat="server" CssClass="input-large mandatory" TextMode="Password" placeholder="Password" />
		</div>
	</div>
</div>
<div class="control-group bordertop thick">
	<asp:LinkButton ID="btnSumbitSimpleForm" runat="server" CssClass="btn btn-primary">Submit</asp:LinkButton>
	<asp:LinkButton ID="btnCancelSimpleForm" runat="server" CausesValidation="false" CssClass="btn btn-inverse">Cancel</asp:LinkButton>
</div>

Handling errors

There are several ways to display error to the user following an unsuccessfull form submission.

Form validation errors

When form fails validation the field the message should appear right next to the field that is invalid. The validator will use a class alerts which will align with any field inside of the controls container.

Login cannot be blank
Password cannot be blank 
<div class="form-horizontal">
	<div class="control-group">
		<label class="control-label" for="<%=txtLogin.ClientID%>">Login</label>
		<div class="controls">
			<asp:TextBox ID="txtLogin" runat="server" CssClass="input-large mandatory" placeholder="Login" />
			<asp:RequiredFieldValidator ID="rfvLogin" runat="server" CssClass="alerts" ControlToValidate="txtLogin" ErrorMessage="Login field cannot be empty!" Display="Dynamic"></asp:RequiredFieldValidator>
		</div>
	</div>
	<div class="control-group">
		<label class="control-label" for="<%=txtPassword.ClientID%>">Password</label>
		<div class="controls">
			<asp:TextBox ID="txtPassword" runat="server" CssClass="input-large mandatory" TextMode="Password" placeholder="Password" />
			<asp:RequiredFieldValidator ID="rfvPassword" runat="server" CssClass="alerts" ControlToValidate="txtPassword" ErrorMessage="Password field cannot be empty!" Display="Dynamic"></asp:RequiredFieldValidator>
		</div>
	</div>
</div>

<div class="control-group bordertop thick">
	<asp:LinkButton ID="btnSubitValidatedForm" runat="server" CssClass="btn btn-primary">Submit</asp:LinkButton>
	<asp:LinkButton ID="btnCancelValidatedForm" runat="server" CausesValidation="false" CssClass="btn btn-inverse">Cancel</asp:LinkButton>
</div>

Form errors

When you need to display an error to the user that is not a validation error this can be handled using built in alert component. Simply add a container with the class alert to the top of the form and trigger the display server side
